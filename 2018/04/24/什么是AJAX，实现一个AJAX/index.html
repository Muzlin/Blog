AJAX(Asynchronous JavaScript and XML) 异步的 JavaScript 和 XML

HTML页面是如何发送请求的

  • 用form表单发送请求

form表单可以发送GET POST请求
缺点：会刷新页面或者新开窗口

  • 用a标签发送请求

a标签可以发送GET请求
缺点：会刷新页面或者新开窗口

  • 用IMG标签发送请求

img标签可以发送GET请求
缺点：只能以图片的形式展示

  • 用LINK标签发送请求

link标签可以发送GET请求
缺点：只能以 CSS、favicon 的形式展示

  • 用SCRIPT标签发送请求

script标签可以发送GET请求
缺点：只能以脚本的形式运行

？有没有什么方式可以实现：

  1. get、post、put、delete 请求都行
  2. 想以什么形式展示就以什么形式展示

微软的贡献

IE5率先在JS中引入ActiveX对象（API），使得JS可以直接发起HTTP请求。
随后Mozilla、Safari、Opera也跟进（抄袭）了，取名XMLHttpRequest，并被纳入W3C规范

AJAX的出现

Jesse James Garrett 讲如下技术取名叫做 AJAX：异步的 JavaScript 和 XML

  1. 使用 XMLHttpRequest 发请求
  2. 服务器返回XML格式数据
  3. JS解析XM数据并更新局部页面

JSON 一门新的语言

AJAX 顾名思义，服务器返回的数据格式是XML的格式。因为在当时，XML是一种比较亲和好处理的数据格式，后来技术的发展，道格拉斯发明(抄袭JS)了一门新的语言JSON。是AJAX(J)有了很大的改进，大家可以浏览JSON官网查看更多的信息。

同源策略

只有 协议+域名+端口 一模一样才允许发 AJAX 请求

一模一样 * 3

lee.com 跟 www.lee.com 不一样
http://lee.com 跟 https://lee.com 不一样
lee.com:6666 跟 lee.com:8888 不一样

浏览器必须保证
只有 协议+域名+端口 一模一样才允许发 AJAX 请求

跨域 CORS

突破同源策略 === 跨域

只需要服务器设置允许指定的不同源的网站访问就叫跨域

这样我的网站lee.com就可以AJAX请求wuyanzu.com的接口了
